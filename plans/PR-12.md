## Content Bundle Contract (Decision Complete)

### File discovery
- Recursively scan the provided root directory
- Include files ending with `.md` or `.mdx`
- Ignore all other files

### One file = one section draft version document
Each markdown file represents exactly one **section version draft**.

### Frontmatter schema (required/optional)
Use `gray-matter` frontmatter and validate with `zod`.

Required fields:
- `path_slug: string`
- `path_title: string`
- `module_slug: string`
- `module_title: string`
- `section_slug: string`
- `section_title: string`
- `version_number: number` (integer, `>= 1`)

Optional fields (with defaults):
- `path_description?: string | null` -> default `null`
- `path_sort_order?: number` -> default `0`
- `module_description?: string | null` -> default `null`
- `module_sort_order?: number` -> default `0`
- `section_sort_order?: number` -> default `0`
- `section_has_quiz?: boolean` -> default `false`
- `change_log?: string | null` -> default `null`
- `created_by?: string | null` -> default `null`
- `estimated_seconds?: number | null` -> default `null`

### Body parsing semantics (PR-12)
- The file body is parsed into a **single** lesson block:
  - `blockType = "markdown"`
  - `blockOrder = 1`
  - `contentJson = { markdown: <raw body string trimmed of leading/trailing whitespace> }`
  - `estimatedSeconds = frontmatter.estimated_seconds ?? null`
- `.mdx` files are treated the same as `.md` (raw text preserved in markdown block)
- No AST parse and no JSX validation in PR-12

### Draft-only enforcement
- Imported `sectionVersion.status` is always normalized to `draft`
- If frontmatter contains `status`, emit a warning and ignore it

## Parser Behavior (Decision Complete)

### `parseContentBundle(rootPath)`
Implement in `/Users/poski/academy/packages/content-importer/src/importer.ts`.

Steps:
1. Resolve absolute root path
2. Recursively discover `.md/.mdx` files
3. Parse each file:
   - read file text
   - parse frontmatter/body via `gray-matter`
   - validate frontmatter
   - normalize document into path/module/section/sectionVersion draft objects
4. Aggregate normalized entities with de-duplication by slug/key
5. Emit validation messages for conflicts/invalid docs
6. Return `ImportParseReport`

### De-duplication keys
- Path: `path.slug`
- Module: `module.slug`
- Section: `section.slug`
- SectionVersion: `${sectionSlug}:${versionNumber}`

### Conflict handling rules
If duplicate keys appear with mismatched metadata:
- Emit `error`
- Keep the first parsed entity
- Skip conflicting duplicate from aggregates

Examples:
- Same `path_slug` but different `path_title`
- Same `module_slug` but different `path_slug`
- Same `section_slug` but different `module_slug`
- Same `section_slug + version_number` repeated in two files

### Invalid file handling
- Invalid frontmatter or unreadable parse => emit `error`, skip file
- Empty markdown body => emit `warning` but still include document with empty markdown string
- Missing required fields => emit `error`, skip file

### Report ordering (for stable output/tests)
Sort arrays before returning:
- `paths` by `slug`
- `modules` by `(pathSlug, slug)`
- `sections` by `(moduleSlug, slug)`
- `sectionVersions` by `(sectionSlug, versionNumber)`
- `messages` by `(sourcePath ?? '', level, code, message)`

## CLI Plan (`/Users/poski/academy/packages/content-importer/src/cli.ts`)

### Command contract
PR-12 CLI is dry-run only and prints JSON report.

Usage:
- `pnpm --filter @academy/content-importer import -- --root <path>`
- Optional `--pretty` to pretty-print JSON (default true in PR-12 is acceptable; choose one and document)

Flags:
- `--root <path>` (required)
- `--pretty` (optional boolean, default `true`)
- `--strict` (optional; if any errorCount > 0, exit code `1`; otherwise `0`)

Default exit codes:
- `0` for successful parse run (even with warnings)
- `1` for CLI usage error
- `1` in `--strict` mode when `errorCount > 0`

### Output
Print exactly one JSON object matching `ImportParseReport` to stdout.

No DB/apply flags yet (deferred to PR-13).
