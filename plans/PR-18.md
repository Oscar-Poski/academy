# PR-18 Plan: Short-Answer Scoring + Quiz Read Endpoints (`latest`, `result`)

## Summary
PR-18 extends the existing quiz submit API from PR-17 to support short-answer grading and adds two read endpoints:

- `GET /v1/quizzes/sections/:sectionId/attempts/latest`
- `GET /v1/quizzes/sections/:sectionId/result`

Goals:
- Grade both `mcq` and `short_answer` questions in a single submission.
- Persist attempt/answer data with per-question correctness across both types.
- Expose latest attempt payload and pass-state summary for gating/UI.
- Keep PR compact (no schema migration, no web changes).

## Public API / Contract Changes

## 1) Existing submit endpoint (expanded)
`POST /v1/quizzes/sections/:sectionId/attempts`

### Request (additive)
```ts
interface QuizSubmissionDto {
  answers: Array<{
    question_id: string;
    selected_option?: string; // for mcq
    answer_text?: string;     // for short_answer
  }>;
}
```

Backward compatibility:
- Existing PR-17 payloads with only `selected_option` continue to work for MCQ.

### Response (expanded feedback)
```ts
interface QuizAttemptResultDto {
  attemptId: string;
  userId: string;
  sectionId: string;
  sectionVersionId: string;
  attemptNo: number;
  score: number;
  maxScore: number;
  passed: boolean;
  submittedAt: string;
  feedback: Array<{
    questionId: string;
    questionType: 'mcq' | 'short_answer';
    isCorrect: boolean;
    awardedPoints: number;
    expectedOption: string | null;        // mcq only
    acceptedAnswers: string[] | null;     // short_answer exact/exact_ci
    expectedPattern: string | null;       // short_answer regex
    selectedOption: string | null;        // mcq only
    answerText: string | null;            // short_answer only
    explanation: string | null;
  }>;
}
```

## 2) New latest-attempt endpoint
`GET /v1/quizzes/sections/:sectionId/attempts/latest` (requires `x-user-id`)

Response:
- `200` + `QuizAttemptResultDto` for latest attempt by `submittedAt DESC, attemptNo DESC`.
- `404` if user has no attempts for that section.
- `400` missing/unknown user.
- `404` section not found.

---

## 3) New quiz-result endpoint
`GET /v1/quizzes/sections/:sectionId/result` (requires `x-user-id`)

Response:
```ts
interface QuizResultDto {
  sectionId: string;
  hasAttempt: boolean;
  latestAttempt: {
    attemptId: string;
    attemptNo: number;
    sectionVersionId: string;
    score: number;
    maxScore: number;
    passed: boolean;
    submittedAt: string;
  } | null;
}
```

Behavior:
- `200` always when section exists.
- If no attempt: `hasAttempt=false`, `latestAttempt=null`.
- If attempt exists: summary from latest attempt.
- `400` missing/unknown user.
- `404` section not found.

## Scoring & Validation Rules (locked)

## Question set
- Score **all** questions for resolved section version where type in `{mcq, short_answer}`.
- If no scorable questions, `400` (`No scorable questions available for this section version`).

## Section version resolution
Reuse PR-17 logic exactly:
- progress-pinned published/archived version first
- else latest published

## Submission matching
- Deduplicate by `question_id`; duplicates => `400`.
- Unknown `question_id` in payload => `400`.
- Omitted question => treated as unanswered and incorrect.

## MCQ grading
- expected from `answerKeyJson.correct_option` string.
- submitted from `selected_option`.
- correct if trimmed exact match (case-sensitive).

## Short-answer grading
`answerKeyJson.mode` controls grading:

1. `exact`
- `answerKeyJson.accepted: string[]` required.
- correct if trimmed submitted text exactly equals one accepted value (case-sensitive).

2. `exact_ci`
- `accepted: string[]` required.
- compare case-insensitive on trimmed values.

3. `regex`
- `answerKeyJson.pattern: string` required.
- optional `answerKeyJson.flags: string` (default empty).
- compile `RegExp(pattern, flags)`.
- correct if regex matches submitted trimmed answer text.

Misconfiguration handling:
- Invalid/missing mode config => `500` (server data issue).
- Invalid regex pattern/flags => `500`.

## Pass threshold
- unchanged: `score / maxScore >= 0.7`.

## Persistence
- One `quiz_attempts` row per valid submission.
- One `quiz_attempt_answers` row per scorable question (including unanswered).
- `answer_json` shape:
  - MCQ: `{ selected_option: string | null }`
  - Short-answer: `{ answer_text: string | null }`
- `grading_details_json` stores full computed feedback array.
