## New model: `Question`
Table: `questions`

Fields:
- `id String @id @default(cuid())`
- `sectionVersionId String @map("section_version_id")`
- `blockId String? @map("block_id")` (nullable)
- `type QuestionType`
- `prompt String`
- `optionsJson Json? @map("options_json")` (for MCQ)
- `answerKeyJson Json @map("answer_key_json")`
- `explanation String?`
- `points Int @default(1)`
- `sortOrder Int @default(0) @map("sort_order")`
- `createdAt DateTime @default(now()) @map("created_at")`

Relations:
- `sectionVersion SectionVersion @relation(fields: [sectionVersionId], references: [id], onDelete: Cascade)`
- `block LessonBlock? @relation(fields: [blockId], references: [id], onDelete: SetNull)`
- `attemptAnswers QuizAttemptAnswer[]`

Indexes/constraints:
- `@@index([sectionVersionId, sortOrder])`
- `@@index([blockId])`
- `@@map("questions")`

## New model: `QuizAttempt`
Table: `quiz_attempts`

Fields:
- `id String @id @default(cuid())`
- `userId String @map("user_id")`
- `sectionId String @map("section_id")`
- `sectionVersionId String @map("section_version_id")`
- `attemptNo Int @map("attempt_no")`
- `submittedAt DateTime @default(now()) @map("submitted_at")`
- `score Int`
- `maxScore Int @map("max_score")`
- `passed Boolean`
- `gradingDetailsJson Json @map("grading_details_json")`

Relations:
- `user User @relation(fields: [userId], references: [id], onDelete: Cascade)`
- `section Section @relation(fields: [sectionId], references: [id], onDelete: Cascade)`
- `sectionVersion SectionVersion @relation(fields: [sectionVersionId], references: [id], onDelete: Cascade)`
- `answers QuizAttemptAnswer[]`

Indexes/constraints:
- `@@unique([userId, sectionId, attemptNo])`
- `@@index([userId, sectionId, submittedAt])`
- `@@index([sectionVersionId])`
- `@@map("quiz_attempts")`

## New model: `QuizAttemptAnswer`
Table: `quiz_attempt_answers`

Fields:
- `id String @id @default(cuid())`
- `attemptId String @map("attempt_id")`
- `questionId String @map("question_id")`
- `answerJson Json @map("answer_json")`
- `isCorrect Boolean @map("is_correct")`
- `awardedPoints Int @default(0) @map("awarded_points")`

Relations:
- `attempt QuizAttempt @relation(fields: [attemptId], references: [id], onDelete: Cascade)`
- `question Question @relation(fields: [questionId], references: [id], onDelete: Cascade)`

Indexes/constraints:
- `@@unique([attemptId, questionId])`
- `@@index([attemptId])`
- `@@index([questionId])`
- `@@map("quiz_attempt_answers")`

## Seed Plan (`/Users/poski/academy/apps/api/prisma/seed.ts`)

Add a helper after section versions are upserted:
- find `request-response-cycle` published version (`version_number = 1`)
- replace questions for that version idempotently:
  - `deleteMany({ where: { sectionVersionId } })`
  - `createMany(...)` with at least 3 questions:
    - 2 `mcq`
    - 1 `short_answer`
  - deterministic `sortOrder` (1..N)
  - meaningful `answerKeyJson` shapes for upcoming scoring service

Example seeded shapes:
- MCQ:
  - `optionsJson: { options: [...] }`
  - `answerKeyJson: { correct_option: "..." }`
- Short answer:
  - `optionsJson: null`
  - `answerKeyJson: { accepted: ["..."], mode: "exact_ci" }`

No seed data for attempts/answers in PR-16.

Idempotency target:
- rerunning seed does not duplicate questions.
