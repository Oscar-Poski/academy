## 2) Gamification service behavior
Create `/Users/poski/academy/apps/api/src/modules/gamification/gamification.service.ts` with:

### Public methods
1. `getSummary(userId: string): Promise<GamificationSummaryDto>`
2. `awardSectionCompleteXp(userId: string, sectionId: string, tx?: Prisma.TransactionClient): Promise<void>`
3. `awardQuizPassXp(userId: string, sectionId: string, attemptId: string, tx?: Prisma.TransactionClient): Promise<void>`

### Internal helpers
1. `assertKnownUser(userId: string): Promise<void>`
2. `computeLevel(totalXp: number): number`
3. `awardXpEvent(input, tx?): Promise<void>`

### XP award semantics (locked)
1. Section-complete XP idempotency key:
- `section_complete:${userId}:${sectionId}`
2. Quiz-pass XP idempotency key:
- `quiz_pass:${userId}:${sectionId}`

This enforces “once per user+section per rule,” regardless of retries or repeated pass attempts.

### Default XP values (locked)
Use env-overridable constants with safe fallbacks:
1. `XP_SECTION_COMPLETE` default `50`
2. `XP_QUIZ_PASS` default `100`

### Level formula (locked)
- `level = Math.floor(totalXp / 100) + 1`
- minimum level is `1`

### Award transaction behavior
`awardXpEvent` runs in one transaction (using provided `tx` when available):
1. Create `xp_events` row with unique `idempotency_key`.
2. Upsert `user_levels` row:
- create with `totalXp = xpDelta`, `level = computeLevel(xpDelta)`
- update with `totalXp += xpDelta`, recompute level
3. On unique violation for `idempotency_key` (`P2002`), treat as no-op success (idempotent replay).
