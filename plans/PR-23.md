# PR-23 Plan: Web Locked States + Lock Messaging (Consume PR-22 Content Lock Metadata)

## Summary
PR-23 makes lock state visible and actionable in the web UI using the additive lock metadata shipped in PR-22.

Goals:
- Render locked badges and disable navigation links for locked modules/sections.
- Show clear lock reasons in path/module/player contexts.
- Keep existing pages resilient when lock metadata is absent (anonymous/unknown user context).

Out of scope:
- No API/backend changes.
- No schema/migration changes.
- No unlock evaluation trigger in UI (still manual via API for now).
- No completion gating behavior changes.

## UI Behavior Decisions (Locked)

## 1) Path Page (`/paths/[pathId]`)
File: `/Users/poski/academy/apps/web/app/paths/[pathId]/page.tsx`

- Fetch path with user context enabled:
  - `getPath(pathId, { includeUserContext: true })`
- For each module card:
  - if `module.lock?.isLocked === true`:
    - show lock badge in header: `Locked`
    - show first reason line under module title
    - replace `Open Module` link with disabled text/button-like span
- For module’s section list:
  - if `section.lock?.isLocked === true`:
    - render non-clickable row + lock badge
  - else render existing `<Link>` row

## 2) Module Page (`/modules/[moduleId]`)
File: `/Users/poski/academy/apps/web/app/modules/[moduleId]/page.tsx`

- Fetch module with user context enabled:
  - `getModule(moduleId, { includeUserContext: true })`
- In module header:
  - if `module.lock?.isLocked === true`, show lock badge + first reason
- Section list:
  - locked sections become non-clickable rows with badge and reason hint
  - unlocked sections remain clickable links
- Existing progress badges remain; lock badge is additive and visually distinct.

## 3) Learn Page Data Wiring (`/learn/[sectionId]`)
File: `/Users/poski/academy/apps/web/app/learn/[sectionId]/page.tsx`

- Keep `getSection(..., { includeUserContext: true })` as-is.
- Fetch module/path with user context enabled:
  - `getModule(section.moduleId, { includeUserContext: true })`
  - `getPath(module.pathId, { includeUserContext: true })`
- This enables lock metadata in player sidebar and nav.

## 4) Player Sidebar Locked States
File: `/Users/poski/academy/apps/web/src/components/player/PlayerSidebar.tsx`

- Module link:
  - locked module -> render non-clickable label + `Locked` badge + optional short reason.
- Section links:
  - locked section -> render non-clickable row.
  - active current section remains visually active and clickable only if unlocked metadata does not mark it locked.
- Keep existing tree structure/order unchanged.

## Error/Fallback Behavior

1. If lock metadata absent:
- treat items as unlocked.
- preserve current clickable behavior.

2. If user context unavailable (`NEXT_PUBLIC_TEMP_USER_ID` missing):
- content pages still render.
- lock metadata omitted.
- no crashes introduced by PR-23 fetches.

3. If lock reason list empty for locked item:
- show generic fallback label: `Locked`.
