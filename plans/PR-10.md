## Important Public API / Interface Additions

### New Endpoint
- `POST /v1/analytics/events`

### Request Body (single event, snake_case)
```ts
interface IngestAnalyticsEventDto {
  event_name: string;
  occurred_at: string; // ISO timestamp (required)
  user_id?: string | null;
  path_id?: string | null;
  module_id?: string | null;
  section_id?: string | null;
  section_version_id?: string | null;
  payload_json?: unknown; // defaults to {}
}
```

### Response Body
```ts
interface IngestAnalyticsEventResponseDto {
  id: string;
}
```

### Error Behavior (PR-10 baseline)
- `400` if `event_name` is missing/blank
- `400` if `occurred_at` is missing or invalid date string
- `201` on successful persistence

Notes:
- No idempotency key support yet (PR-11)
- No strict event name enum yet (PR-11)
- No `x-user-id` header requirement for analytics ingest in PR-10 (raw event body is authoritative)

## Prisma Schema Changes (`analytics_events`)

Add a new model to `/Users/poski/academy/apps/api/prisma/schema.prisma`:

```prisma
model AnalyticsEvent {
  id               String   @id @default(cuid())
  userId           String?  @map("user_id")
  eventName        String   @map("event_name")
  pathId           String?  @map("path_id")
  moduleId         String?  @map("module_id")
  sectionId        String?  @map("section_id")
  sectionVersionId String?  @map("section_version_id")
  payloadJson      Json     @map("payload_json")
  occurredAt       DateTime @map("occurred_at")
  receivedAt       DateTime @default(now()) @map("received_at")

  @@index([eventName, occurredAt])
  @@index([userId, occurredAt])
  @@index([sectionId, occurredAt])
  @@index([moduleId, occurredAt])
  @@map("analytics_events")
}
```

### Schema Design Decisions (Locked In)
- `payload_json` is required in DB, but endpoint defaults it to `{}` if omitted.
- No foreign keys to `users/paths/modules/sections/section_versions` in PR-10.
  - Reason: this is a raw event log; ingestion should not fail if referenced entities are missing/deleted.
- `received_at` is server-generated (`now()`), while `occurred_at` is client-supplied and required.

- returns `Promise<IngestAnalyticsEventResponseDto>`

No header parsing in PR-10.

### Service (`analytics.service.ts`)
Add `ingestEvent(body)` method.

Implementation steps (decision complete):
1. Validate `event_name` is non-empty string after trim
2. Validate `occurred_at` parses to a valid `Date`
3. Build normalized create payload:
   - trim `event_name`
   - coerce missing optional IDs to `null`
   - `payloadJson = body.payload_json ?? {}`
   - `occurredAt = parsedDate`
4. `prisma.analyticsEvent.create(...)`
5. Return `{ id: created.id }`

### Validation Style
Use manual validation (`BadRequestException`) in service (same style as `ProgressService`) rather than adding class-validator/pipes in this PR.

## Explicit Assumptions and Defaults

- PR-10 accepts a **single event per request** (no batch ingest yet).
- Event payload contract uses **snake_case request fields** for consistency with existing progress request DTOs.
- Analytics ingest is intentionally **permissive** in PR-10 (raw string `event_name`, no enum restrictions, no idempotency).
- `user_id` and content IDs are stored as nullable scalar columns without foreign keys to preserve raw log ingest reliability.
- `payload_json` defaults to `{}` when omitted.
